No img alt text (images are CSS/JS injected)
All car images in the main SPA (Lamborghinis, Ferraris, before/afters) are injected by the compiled React bundle as CSS background-image or dynamic elements — there are no <img> tags in the HTML. Alt text would require a rebuild of the JS bundle with proper <img alt="..."> markup. This is the last significant unfixed item.
Impact: HIGH — 4 pts · Fix: Rebuild bundle with proper img tags
📝
SPA service pages have no crawlable body text (partially mitigated)
The 8 SPA shell pages (/services, /ceramic-coating, etc.) now have noscript blocks with keyword-rich text. However, Googlebot typically renders JavaScript, so this mitigates but doesn't fully solve the crawlable-content issue for JS-disabled crawlers. The per-route meta updater ensures correct metadata for all routes.
Impact: MEDIUM — 2 pts · Mitigation: noscript blocks added (Round 5)
